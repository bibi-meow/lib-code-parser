<objective>
Introduce the language dimension into dispatch ONCE (D-01), the load-bearing structural change every C++ extractor depends on. Nest `PRIMITIVES` and `EVALUATIONS` into `dict[language, dict[name, fn]]`, migrate existing Python registrations under `["python"]` (values byte-unchanged), and change the executor's two walk lines to index by `cav.language`. This is the ONLY plan in Phase 4 that modifies an existing executor/dispatch file.

Purpose: The flat dispatch contract is structurally incapable of hosting a second language without violating Open-Closed invariants #1/#2/#4 or breaking DET-01/LNG-04 (CONTEXT.md Decision A). Nesting resolves the locked-contract collision so cpp extractors run ONLY on cpp CAV and output slot names stay common across languages (natural LNG-04 parity).
Output: Nested dispatch dicts, cav.language-indexed executor walks, docs/09 language-dimension procedure, updated dispatch tests.
</objective>
